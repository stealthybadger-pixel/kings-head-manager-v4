## ThermoPro TP25H2 Bluetooth meat thermometer — DONE, shipped
- Reverse-engineered + verified against the device and the `daniel-corbett/thermopro-cli`
  source (decompiled from the official Android app). Protocol:
  service `1086fff0-3343-4817-8bb2-b32206336ce8`, write char FFF1, notify char FFF2.
  Sequence: subscribe FFF2 → wait 0.5s → write 12-byte handshake
  `01 09 8a 7a 13 b7 3e d6 8b 67 c2 a0` to FFF1 → device streams `0x30` frames ~1/sec.
  Decode: battery=byte2; probes at byte offsets 5-6/7-8/9-10/11-12, 2-byte custom BCD
  (sign bit 0x80, hundreds (b1&0x70)>>4, tens b1&0x0F, ones (b2&0xF0)>>4, tenths b2&0x0F);
  `ff ff` = probe unplugged. Full write-up in repo: `thermopro_protocol_guide.md`;
  standalone capture tool: `scripts/ble_probe_sniffer.html`.
- Built `src/hooks/useBleThermometer.ts` (mirrors useBleScale): connect, handshake,
  keepalive re-send every 5s, auto-reconnect around the device's ~15-20s idle drop.
  Wired into Food Temp Checks modal — tap a live probe reading to fill the temp field,
  user still presses Record (never auto-saved). Probe slots labelled **"Raw Meat" /
  "Cooked Item"** (first/second plugged port, in slot order — flip if it reads swapped).
- **Silent reconnect after refresh** via `navigator.bluetooth.getDevices()` (no chooser).
  Plus clean `gatt.disconnect()` on pagehide/beforeunload so the probe frees its single
  connection slot and keeps advertising — this removed the need to power-cycle it after a
  refresh. Confirmed working live by the user.
- **DEVICE SETUP REQUIREMENT (non-obvious, operational):** silent reconnect needs the
  Chrome flag `chrome://flags/#enable-web-bluetooth-new-permissions-backend` = Enabled,
  once per device/browser. Without it you get the chooser dialog on every refresh. Must be
  enabled on the **Android tablet** and the **MacBook** (both Chrome). **iOS is NOT
  supported at all** (Apple disables Web Bluetooth in Safari/Chrome) — would need the
  Bluefy browser app. A one-time dismissible in-app hint (localStorage key
  `probeFlagHintDismissed`) points users to the flag.
